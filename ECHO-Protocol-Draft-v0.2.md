## 1. 核心概念

### 1.1 四权力框架
数字资产的四种基本权利：

| 权力 | 含义 | 元亨利贞对应 |
|:----:|------|:------------:|
| **用** | 存在方式（怎么用） | **元** - 创始根源 |
| **扩** | 连接范围（谁能用） | **亨** - 通达顺畅 |
| **衍** | 衍生能力（能改吗） | **和** - 和谐生化 |
| **益** | 代谢模式（稀缺吗） | **贞** - 正固守恒 |

> **v0.2 修正**："衍"从"利-获利"改为"和-和谐"，更符合《文言》"利者，义之和也"的本义。

### 1.2 权力依赖关系（v0.2 新增）

```
用（元）→ 扩（亨）→ 衍（和）→ 益（贞）
  ↓
用 = 禁(0) 时，扩 = 独(0)，衍 = 止(0)，自动锁定
```

**设计原理**：没有使用权（用），其他权利无从谈起。这是权利层级的真实表达。

### 1.3 基因码编码
12位二进制编码：
```
基因码 = (用 << 9) | (扩 << 6) | (衍 << 3) | 益
范围：0 - 4095 (2^12)

v0.2 规划：扩展至 128bit
- 前 12bit：权力基因（保留 ECHO 特征）
- 后 116bit：随机唯一标识（防碰撞）
```

### 1.4 卦象生成算法

**v0.2 修正：先天八卦数序**

传统先天八卦：1乾、2兑、3离、4震、5巽、6坎、7艮、8坤

编程索引（0-based）：
```
0:坤(地)  1:艮(山)  2:坎(水)  3:巽(风)
4:震(雷)  5:离(火)  6:兑(泽)  7:乾(天)
```

> **关键修正**：0=坤(全阴)，7=乾(全阳)，符合二进制 000→111 的自然递进。

**卦象计算**：
```
上卦（天道）= (用 + 扩) mod 8  → 创造与传播的合力
下卦（地道）= (衍 + 益) mod 8  → 生长与回归的循环
卦序        = 上卦 × 8 + 下卦 + 1  → 1-64
```

**未来改进（梅花易数）**：
```
上卦 = 用 mod 8          (天道)
下卦 = 扩 mod 8          (地道)
动爻 = (衍 + 益) mod 6   (人道，哪一爻变)
```

### 1.5 气数算法（v0.2 重大升级）

**v0.1（已废弃）**：`qi = |sin(基因码 × 时间 × φ)|`
- 问题：可预测、可操控、无混沌性

**v0.2（当前）**：Logistic 混沌映射
```javascript
// Logistic 混沌映射：x(n+1) = r × x(n) × (1 - x(n))
// 当 r = 3.99 时，系统进入混沌状态

function calculateQi(geneCode, timestamp) {
  const r = 3.99;  // 混沌临界值
  
  // 初始值由基因码决定，确保不同资产有不同起点
  let x = (geneCode % 9973) / 9973;
  if (x === 0) x = 0.618;  // 黄金比例作为备用
  
  // 迭代次数基于时间戳，使气数随时间流转
  const iterations = 50 + (timestamp % 50);
  
  // 混沌迭代：蝴蝶效应，微小差异导致巨大结果差异
  for (let i = 0; i < iterations; i++) {
    x = r * x * (1 - x);
  }
  
  return x;  // 0-1 之间
}
```

**混沌特性**：
| 特性 | 效果 |
|------|------|
| **确定性** | 相同输入 → 相同输出（可重现） |
| **敏感性** | 基因码差1，气数天差地别 |
| **不可预测** | 长期行为无法推算 |
| **时变性** | 随时间自然流转 |

> **为什么用混沌？** "气"的概念正是：确定性系统中的不确定性，周而复始的自然节律。Logistic 映射是描述这种"有序中的无序"最经典的数学模型。

**气数等级**：
- 0.0-0.25: dormant（沉寂）
- 0.25-0.5: gentle（温润）
- 0.5-0.75: vigorous（蓬勃）
- 0.75-1.0: fierce（凛冽）

### Round 2: 数学验证（密码学家）

| 问题 | 评估 | 建议 |
|------|:----:|------|
| 12bit基因码 | 🔴 太弱 | 4096种组合，100资产即有67%冲突概率 → **扩展至128bit** |
| 气数算法（旧） | 🔴 可预测 | sin×φ 可被操控 → **已升级为 Logistic 混沌** |
| 气数算法（新） | 🟡 中等安全 | 混沌比 sin 强，但仍不如 CSPRNG → v1.0 考虑 ChaCha20 |
| 黄金比例 φ | 🟢 装饰性 | 无密码学价值，保留作为文化元素 |

**核心反馈**：v0.2 的混沌升级是正确的方向，但基因码长度和更安全的随机数生成仍需在 v1.0 解决。

### Round 3: 工程验证（区块链开发者）

| 组件 | 推荐方案 | 理由 |
|------|----------|------|
| 防作弊 | 平台背书证明 | ECDSA签名 + 单日限流 + 衰减机制 |
| 权利上链 | ERC-721扩展 | 四权力编码上链，支持转让/分割 |
| 跨平台 | The Graph + Chainlink | 去中心化索引 + 预言机对账 |
