# 用户偏好技能库

> 基于与用户的合作经验总结，用于指导未来的开发工作

---

## 1. 设计理念

### 核心原则
- **"设计风格统一 ≠ 布局统一"**
  - 保持视觉语言一致（颜色、字体、圆角、阴影）
  - 但每个页面可以有独特的布局结构
  
- **仪式感 > 功能性装饰**
  - 过渡页要让用户"停下来理解概念"
  - 不是为了好看而加特效
  
- **克制的高级感**
  - 喜欢：玻璃拟态、柔和光晕、呼吸动画
  - 讨厌：花哨、过度设计、炫技式动画

### 视觉元素优先级
1. 氛围光效（背景渐变）
2. 卡片质感（玻璃拟态）
3. 微交互动画（hover、入场）
4. 装饰性元素（角标、边框）

---

## 2. 文案风格

### 沟通原则
- **比喻优先**
  - 复杂概念用生活化比喻
  - 例："数字种子"、"融合成树"、"根属于你，叶子可以借给别人"
  
- **拒绝术语堆砌**
  - 不要说："五维权利独立确权"
  - 要说："权属蓝图"
  
- **一句话讲清**
  - 每段不超过3句话
  - 去掉废话和客套话

### 文案检查清单
- [ ] 是否用了比喻？
- [ ] 是否避免了术语？
- [ ] 能否再精简？
- [ ] 用户能一眼看懂吗？

---

## 3. 技术实现

### 代码哲学
- **改动最小化原则**
  - 不破坏原有格式和布局
  - 新增代码与旧代码隔离
  
- **复用现有系统**
  - 优先使用已有的CSS类
  - 例：`.glass-card`、`.stage-0-content`
  
- **内联样式优先**
  - 简单场景直接用 `style=""`
  - 避免创建过多新CSS类
  
- **渐进增强**
  - 第一步：保证能用（不报错、不重叠）
  - 第二步：优化美观

### CSS策略
```css
/* 推荐：修改现有类 */
.glass-card {
  /* 原有样式 */
}
.glass-card--ceremony {
  /* 新增modifier */
}

/* 避免：创建大量新类 */
.transition-card { }
.transition-title { }
.transition-btn { }
```

## 4. 沟通方式

### 反馈偏好
- **结果导向**
  - 不喜欢听解释
  - 直接说"不行""还是不行"
  
- **明确指令**
  - "退回到稳定版本"
  - "找设计专家"
  - 具体、可执行
  
- **快速迭代**
  - 愿意尝试方案A/B/C
  - 错了立刻止损，不纠结

### 沟通禁忌
- ❌ 长篇解释为什么出错
- ❌ 绕弯子说话
- ❌ 坚持错误方案
- ❌ 过度道歉

---

## 5. 决策模式

### 验证流程
1. **视觉验证**
   - 必须看到实际页面效果
   - 截图反馈
   
2. **敢于放弃**
   - 复杂设计如果出bug，宁可要简单但能用的
   - 及时止损，不纠结
   
3. **实用主义**
   - 宁简勿繁
   - 宁稳勿炫

### 决策检查清单
- [ ] 能正常打开吗？
- [ ] 有重叠/错位吗？
- [ ] 文案通顺吗？
- [ ] 风格统一吗？
- [ ] 用户能看懂吗？

## 6. 快速参考

### 颜色规范
| 页面 | 主题色 | 用途 |
|------|--------|------|
| 铸造相关 | 青色 #00f0ff | 科技、创造 |
| 融合相关 | 金色 #ffd700 | 价值、成果 |
| 通用背景 | 深色 #020208 | 沉稳、专业 |

### 间距规范
- 卡片内边距：60px - 80px
- 标题间距：20px
- 段落间距：30px
- 按钮间距：30px

### 动画规范
- 入场动画：0.5s - 0.8s
- 呼吸动画：3s - 6s
- 悬停动画：0.3s
- 避免：超过1s的动画

---

## 7. 用户画像

**一句话总结：**
> 追求"克制的高级感"——在保持代码健壮的前提下，用最少的设计元素营造最大的仪式感。

**关键特征：**
- 实用主义者（宁简勿繁）
- 结果导向者（不看过程看结果）
- 决策者（敢于放弃、快速止损）
- 品味者（懂设计、不将就）
