# ECHO 东方美学设计团队 — 需求文档

## 项目背景

ECHO 是一个去中心化的数字版权协议，核心理念是「生长」——不是静态的NFT，而是随使用流转、持续生长的数字生命。

现有网站已具备基础东方美学气质（温润纸张、金色点缀、宋式留白），但需要更高级、更克制、更过目不忘的视觉系统。

---

## 团队构成（4-5人）

### 1. 视觉设计总监 (Visual Design Lead)
**核心职责**：定义整体视觉方向，把控「克制的高级感」

**必备能力**：
- 深耕东方美学（宋瓷、侘寂、禅宗）
- 对「留白」「呼吸感」有极致追求
- 能驾驭极简却不空洞的设计
- 熟悉中西文字体搭配

**参考风格**：
- 原研哉（无印良品）的「空」
- 隈研吾建筑的「负建筑」
- 宋瓷汝窑的釉色与开片

**交付物**：
- 视觉概念板（Mood Board）
- 色彩系统（主色、辅助色、灰阶）
- 字体系统（中文、英文、数字）
- 质感/纹理库

---

### 2. UI 设计师 (UI Designer)
**核心职责**：界面组件、图标、按钮等细节打磨

**必备能力**：
- 0.5px 级别的细节控
- 微交互设计（hover、点击、加载）
- 设计系统构建（Design System）
- 熟悉 Tailwind / CSS 变量

**设计原则**：
- 每个元素都有存在理由
- 没有装饰性的元素
- 线条要「细到刚好能看见」
- 动效要「慢到让人静心」

**交付物**：
- 全套 UI 组件库
- 图标系统（线性、极简）
- 交互动效规范
- 响应式断点设计

---

### 3. 交互设计师 (UX/Interaction Designer)
**核心职责**：用户流程、信息架构、情感化体验

**必备能力**：
- 东方哲学思维（渐进、含蓄、意在言外）
- 叙事性设计（让用户感受到「生长」）
- 声音/触觉等感官设计
- 无障碍设计意识

**关键问题**：
- 如何让「上传作品」变成「播种」的仪式？
- 如何让「购买授权」变成「结缘」的体验？
- 如何让数字生命有「呼吸」的感觉？

**交付物**：
- 用户旅程地图
- 关键场景交互流程
- 情感化触点设计
- 声音/动效情绪板

---

### 4. 动效设计师 (Motion Designer)
**核心职责**：让界面「活」起来，但不喧闹

**必备能力**：
- 物理质感模拟（水波、风动、墨晕）
- 缓动曲线调校（ease-out 要足够长）
- CSS / Lottie / GSAP 实现能力
- 理解「静中有动，动中有静」

**动效原则**：
| 场景 | 时长 | 质感 |
|------|------|------|
| 页面切换 | 1.2s | 像翻书般从容 |
| Hover | 0.8s | 像水面泛起涟漪 |
| 加载 | 无限循环 | 像呼吸般起伏 |
| 成功反馈 | 1.5s | 像墨在水中晕开 |

**交付物**：
- 全局动效规范
- 关键场景动效原型
- 声音设计建议
- 实现代码片段

---

### 5. 字体/文案设计师 (Typographer/Copywriter)
**核心职责**：字体的选择与排版，文字的意境

**必备能力**：
- 中文字体专家（知道什么场景用宋体、楷体、黑体）
- 竖排/横排的东方排版美学
- 极简文案（一句顶十句）
- 诗意表达

**字体策略**：
- 标题：思源宋体 或 方正清刻本悦宋（温润、书卷气）
- 正文：思源宋体 Light（轻盈、透气）
- 英文：Cormorant Garamond（衬线优雅）
- 数字：等宽或旧式数字（不突兀）

**交付物**：
- 字体系统规范
- 字号/字重/行高/字间距表
- 中英混排规则
- 微文案（按钮、提示、空状态）

## 核心设计关键词

| 关键词 | 不要 | 要 |
|--------|------|-----|
| **留白** | 空白的浪费 | 呼吸的空间 |
| **线条** | 粗重的边界 | 0.5px 的微妙分隔 |
| **色彩** | 鲜艳夺目 | 汝窑天青、陶土灰褐 |
| **动效** | 快速切换 | 研墨般的从容 |
| **图标** | 扁平填充 | 线性留白 |
| **质感** | 数码冰冷 | 宣纸、绢布、瓷器 |

## 协作流程建议

### 第一阶段：概念共识（2周）
1. 视觉总监牵头，产出 Mood Board
2. 团队共同评审，确定方向
3. 输出「设计语言文档」

### 第二阶段：系统搭建（4周）
1. UI 设计师搭建组件库
2. 字体设计师完成排版规范
3. 动效设计师制作关键 Demo

### 第三阶段：场景深化（4周）
1. 交互设计师细化用户流程
2. 全团队打磨关键页面
3. 动效/声音集成测试

### 第四阶段：交付优化（2周）
1. 前端实现对接
2. 细节调优（像素级）
3. 文档整理
