# ECHO Protocol UI/UX 设计文档

## 1. 项目概述

### 1.1 设计目标
打造一个专业、现代、易用的 Web3 音乐平台，整合 17+ 分散页面为统一体验。

### 1.2 设计理念
- **专业性**: 体现区块链技术的可靠与透明
- **现代感**: 深色主题配合渐变光效，科技感与音乐美学融合
- **易用性**: 降低 Web3 门槛，让普通用户轻松上手

### 1.3 页面架构

```
ECHO Protocol
├── 用户端 (User)
│   ├── 首页/发现 (discover.html) - 个性化推荐、热门、最新
│   ├── 市场 (music-market.html) - 浏览所有音乐
│   ├── 资产详情 (asset-detail.html) - 音乐详情、试听、购买
│   ├── 购买授权 (license-shop.html) - 三合一授权购买流程
│   ├── 榜单 (charts.html) - 热门排行
│   ├── 我的收藏 (my-favorites.html) - 收藏的音乐
│   └── 我的资产 (my-assets.html) - 已购买的 License
├── 创作者端 (Creator)
│   ├── 仪表盘 (creator-dashboard.html) - 统计概览
│   ├── 上传作品 (mint-music.html) - 生成音乐资产
│   ├── 批量生成 (bulk-mint.html) - 批量上传
│   ├── 收益管理 (creator-revenue.html) - 收益统计
│   └── 作品管理 (rights-management.html) - 授权管理
└── 通用 (Common)
    ├── 个人资料 (profile.html)
    ├── 消息通知 (notifications.html)
    └── 404 错误页 (404.html)
```

## 2. 导航结构

### 2.1 顶部导航栏 (Global Navbar)

**结构**:
```
┌─────────────────────────────────────────────────────────────────┐
│  🎵 ECHO Music    [发现] [市场] [榜单] [生成] [数据]    🌓  中文  [连接钱包] │
│   (Logo)          (主导航)                              (控制区)  (操作区)   │
└─────────────────────────────────────────────────────────────────┘
```

**规范**:
- 高度: 64px (固定定位)
- 背景: `rgba(10, 10, 26, 0.95)` + `backdrop-filter: blur(20px)`
- 下边框: `1px solid rgba(255,255,255,0.1)`
- z-index: 1000
- 最大宽度: 1400px，居中

**响应式**:
- 桌面端: 完整导航
- 平板/移动端: Logo + 汉堡菜单 + 钱包按钮

### 2.2 侧边栏 (Creator Sidebar)

仅创作者中心页面使用，固定在左侧。

```
┌──────────┬─────────────────────────────────────┐
│ 🎵 ECHO  │                                     │
│──────────│                                     │
│ 📊 仪表盘 │         主内容区                    │

│ 📦 批量   │                                     │

│──────────│                                     │
│ ← 返回市场│                                     │
└──────────┴─────────────────────────────────────┘
```

**规范**:
- 宽度: 240px (桌面端)
- 背景: 比主背景稍深的色值
- 可折叠: 平板端收起为图标栏 (72px)

### 2.3 面包屑导航 (Breadcrumb)

**使用场景**: 资产详情页、购买流程等深层页面

```
首页 > 市场 > 电子音乐 > Summer Vibes > 购买授权
```

**规范**:
- 位置: 导航栏下方，页面内容上方
- 字号: 14px
- 颜色: 次级文字色
- 分隔符: `>` 或 `/`
- 当前页: 无链接，主色高亮

## 3. 布局系统

### 3.1 栅格系统

- **12列栅格**: 基于 CSS Grid
- **列间距 (Gutter)**: 24px (桌面) / 16px (平板) / 12px (移动)
- **外边距 (Margin)**: 
  - 桌面: 40px
  - 平板: 24px
  - 移动: 16px

### 3.2 内容容器

```css
.container {
  max-width: 1400px;  /* 大屏 */
  margin: 0 auto;
  padding: 0 20px;
}

.container-narrow {
  max-width: 960px;   /* 阅读型页面 */
}

.container-wide {
  max-width: 1600px;  /* 数据仪表盘 */
}
```

### 3.3 间距系统 (Spacing Scale)

| Token | 值 | 用途 |
|-------|-----|------|
| space-xs | 4px | 图标内边距、紧凑元素间距 |
| space-sm | 8px | 组件内部间距 |
| space-md | 16px | 卡片内边距、表单间距 |
| space-lg | 24px | 区块间距、卡片间距 |
| space-xl | 32px | 大区块分隔 |
| space-2xl | 48px | 页面区块 |
| space-3xl | 64px | 主要区块分隔 |

## 4. 响应式断点

| 断点 | 宽度 | 设备类型 | 导航模式 |
|------|------|----------|----------|
| sm | < 640px | 手机竖屏 | 汉堡菜单 |
| md | 640px - 767px | 手机横屏 | 汉堡菜单 |
| lg | 768px - 1023px | 平板 | 简化导航 |
| xl | 1024px - 1279px | 小笔记本 | 完整导航 |
| 2xl | 1280px+ | 桌面 | 完整导航 + 侧边栏 |

### 4.1 响应式规则

**网格列数**:
- 移动端: 1-2 列
- 平板: 2-3 列
- 桌面: 3-4 列
- 大屏: 4-6 列

**字体缩放**:
- 移动端: 基准 14px
- 平板+: 基准 16px

**间距缩放**:
- 移动端: 间距 × 0.75
- 平板: 间距 × 0.875
- 桌面+: 间距 × 1

## 6. 交互模式

### 6.1 钱包连接流程

```
[点击连接钱包]
      ↓
┌───────────────────┐
│ 选择钱包提供商    │
│ ┌───┐ ┌───┐      │
│ │Meta│ │Wallet│   │
│ │Mask│ │Connect│  │
│ └───┘ └───┘      │
└───────────────────┘
      ↓
[弹出钱包确认]
      ↓
[连接成功，显示地址]
```

### 6.2 购买流程

```
选择授权 → 确认订单 → 钱包签名 → 链上确认 → 购买成功
   │           │           │           │
   ▼           ▼           ▼           ▼
显示价格    显示总价    调用合约    显示NFT/交易哈希
```

### 6.3 上传生成流程

```
填写信息 → 上传文件 → 配置权益 → 设置价格 → 预览确认 → 生成
   │          │          │          │          │
   ▼          ▼          ▼          ▼          ▼
实时验证   进度条    权利卡片    价格卡片   完整预览
```

### 6.4 通用交互规范

**悬停效果**:
- 按钮: `translateY(-2px)` + 阴影增强
- 卡片: 边框发光 + 轻微上浮
- 链接: 颜色变化 + 下划线

**加载状态**:
- 按钮: Spinner + 禁用
- 页面: Skeleton 骨架屏
- 数据: 渐进式加载

**过渡动画**:
- 默认: `transition: all 0.3s ease`
- 页面切换: 淡入淡出 200ms
- 弹窗: 缩放 + 淡入 300ms

**反馈机制**:
- Toast 通知: 右上角滑入
- 成功: 绿色勾选
- 错误: 红色感叹号
- 警告: 黄色三角
- 信息: 蓝色圆点

## 7. 无障碍设计

### 7.1 键盘导航
- 所有交互元素可通过 Tab 访问
- Enter/Space 激活按钮
- Esc 关闭弹窗

### 7.2 屏幕阅读器
- 语义化 HTML 标签
- ARIA 标签标注
- 图片 alt 文本

### 7.3 色彩对比
- 正文对比度 ≥ 4.5:1
- 大文字对比度 ≥ 3:1
- 交互元素对比度 ≥ 3:1

### 7.4 动效减弱
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 13. 动画规范

### 13.1 入场动画
```css
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

### 13.2 交互动画
- 按钮点击: 缩放 0.95
- 卡片悬停: 上浮 + 发光
- 加载: 无限旋转

### 13.3 页面过渡
- 内容区淡入: 300ms
- 列表项错开: 每项延迟 50ms

### 13.4 性能优化
- 使用 `transform` 和 `opacity`
- 添加 `will-change` 提示
- 避免动画触发布局重排
