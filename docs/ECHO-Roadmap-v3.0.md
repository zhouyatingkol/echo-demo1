## 五、风险与应对

### 风险1：6个月上不了线

**应对：**
- MVP砍功能，只保留核心（四权力 + 铸造 + 转移）
- 使用证明系统简化，不上复杂Oracle
- 卦象、气数先前端展示，不上链计算

### 风险2：收入不够生活费

**应对：**
- 第3个月必须开始造血（咨询/课程）
- 同时找外包项目保底
- 必要时暂停，先打工存钱

### 风险3：技术实现有难度

**应对：**
- 我负责技术，有问题我解决
- 复杂功能（如去中心化Oracle）延后
- 先用中心化方案跑起来，再逐步去中心化

### 风险4：用户不买账

**应对：**
- 第1个月就开始找种子用户验证
-  Pivot 成本低，及时调整方向
- 东方美学是差异化，但产品必须好用
