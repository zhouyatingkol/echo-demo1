## 二、双代币体系（实用+稳定）

### 2.1 数字人民币（DCEP）- 价值层

**用途**：
- 创作者出售收益权（用户用 DCEP 购买）
- 平台服务费（高级功能付费）
- 跨链桥接费（如有）

**优势**：
- 合规：完全符合国家法规
- 稳定：无价格波动风险
- 信任：国家信用背书
- 普惠：人人可用，无门槛

**技术实现**：
```
支付方式：
1. 数字人民币钱包扫码支付
2. 银行APP直接转账
3. 第三方支付接入（支付宝/微信 - DCEP通道）

链上记录：
- 支付凭证上链存证（哈希值）
- DCEP流转在链下，ECHO网络只记录"已支付"状态
```

### 2.2 ECHO代币 - 网络层

**定位**：类似 ETH 的 Gas，仅供运行网络

**用途**：
| 场景 | 费用 | 说明 |
|:-----|:-----|:-----|
| **铸造资产** | 0.1 ECHO | 创建新的数字资产 |
| **权力变更** | 0.05 ECHO | 修改四权力配置 |
| **提交使用证明** | 0.01 ECHO | Oracle节点提交数据 |
| **查询历史** | 免费 | 基础查询不收费 |
| **高级分析** | 0.5 ECHO | 复杂数据查询 |

**获取方式**：
1. **兑换**：用 DCEP 在官方渠道兑换（1 ECHO = 0.1 CNY，固定汇率）
2. **使用奖励**：创作者资产被使用，获得少量 ECHO 奖励（网络补贴）
3. **节点运营**：运行 Oracle 节点，获得 ECHO 奖励（服务费）

**关键设计**：
- **固定汇率**：1 ECHO = 0.1 CNY，永不变动
- **无投机空间**：价格锚定，无炒作价值
- **用多少买多少**：按需兑换，不囤积
- **可退款**：未使用的 ECHO 可按原汇率换回 DCEP

### 2.3 为什么不单用 DCEP？

**技术原因**：
- DCEP 是账户模型，不适合高频微支付
- ECHO 作为 Gas，可以精确计量资源使用
- 防止网络 spam（无成本攻击）

**经济原因**：
- 网络运营需要成本（服务器、开发）
- ECHO 费用用于：节点激励、开发维护、生态建设
- 透明可审计：链上可查所有费用流向

**文化原因**：
- ECHO 是网络的"气"，有生命感
- DCEP 是外部的"钱"，是价值交换
- 两者分离：钱归钱，气归气

## 三、创作者经济模型

### 3.1 收益权分式化（用 DCEP）

**案例：小明的《墨竹·风骨》**

```
总收益权 = 10000 份
创作者保留 = 3000 份（30%）
出售 = 7000 份（70%）

每份价格 = 10 DCEP（1元）
首售总额 = 70,000 DCEP = 7,000元

版税机制：
- 每次转售，创作者获得 5% 版税
- 版税用 DCEP 支付
- 链上自动分配
```

**为什么是 DCEP 不是 ECHO？**
- 收益是"价值"，用稳定货币
- ECHO 是"费用"，用实用代币
- 分离后，创作者收益不受 ECHO 价格波动影响

### 3.2 创作者收入构成

| 收入来源 | 货币 | 说明 |
|:---------|:-----|:-----|
| **首售收益** | DCEP | 出售收益权的收入 |
| **版税收入** | DCEP | 每次转售的 5% |
| **使用奖励** | ECHO | 资产被使用的网络补贴 |
| **平台补贴** | DCEP/ECHO | 早期创作者扶持计划 |

### 3.3 平台收入构成

| 收入来源 | 货币 | 用途 |
|:---------|:-----|:-----|
| **铸造费** | ECHO | 节点激励、网络维护 |
| **服务费** | DCEP | 团队运营、开发 |
| **高级功能** | DCEP | 增值服务（数据分析、跨链等） |

## 四、去中心化程度

### 4.1 渐进式去中心化

**阶段一：核心团队主导（0-12个月）**
- 合约升级权：核心团队多签
- 参数调整：团队决策
- Oracle节点：官方运营 + 白名单

**阶段二：社区参与（12-24个月）**
- 引入社区代表参与多签
- Oracle节点开放申请
- 重大决策社区听证

**阶段三：有限自治（24个月+）**
- 核心参数不可更改（锁定）
- 社区提案机制（非 binding）
- 团队保留紧急干预权

### 4.2 为什么不完全DAO？

| 问题 | 东方智慧解决方案 |
|:-----|:---------------|
| **投票率低** | 不依赖投票，依赖"用"（使用即治理） |
| **寡头控制** | 创始人长期持有关键权力，防止资本操控 |
| **决策低效** | 核心团队快速决策，社区提供建议 |
| **治理攻击** | 没有治理代币，无法买票攻击 |

**核心原则**：
> "无为而治"不是"什么都不管"，是"不妄为"。
> 核心团队像园丁：浇水、施肥、修剪，但不强迫花朵怎么开。

## 五、合规与风险管理

### 5.1 监管合规

**中国大陆**：
- 使用 DCEP 作为主要支付：完全合规
- ECHO 作为实用代币：非证券，非虚拟货币炒作
- 不涉及ICO：无公开募资
- 不涉及交易所：ECHO只能在官方渠道兑换

**国际（如有）**：
- 遵循当地法规
- 可能需要 KYC/AML
- 禁止地区不提供

### 5.2 风险控制

| 风险 | 对策 |
|:-----|:-----|
| **政策变化** | 保持透明，主动沟通，拥抱监管 |
| **技术漏洞** | 多重审计，保险基金，紧急暂停 |
| **市场波动** | ECHO锚定DCEP，无价格波动 |
| **竞争替代** | 深耕东方美学，建立文化壁垒 |
| **团队风险** | 多签管理，知识文档化，逐步培养接班人 |
