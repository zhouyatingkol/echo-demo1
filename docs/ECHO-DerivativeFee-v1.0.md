# ECHO 核心机制：衍生费与市场博弈
**用价格信号调节创作生态**

---

## 一、核心洞察

> **"衍生费"不是成本，是信号。**

创作者用衍生费表达自己的价值观：
- 低衍生费 = "欢迎合作，我要的是影响力"
- 高衍生费 = "我的作品很珍贵，要付高价"
- 零衍生费 = "公共财产，随意使用"

---

## 二、衍生费的设定机制

### 2.1 父代资产的定价权

**创建资产时，创作者设定"衍"档位的衍生费：**

```
小明创建《墨竹》：
├── 衍 = 4-混（允许混剪）
└── 衍生费 = 30%（如果有人引用我的作品，我要抽30%）

小红创建《山水》：
├── 衍 = 4-混
└── 衍生费 = 50%（我认为我的作品是主体，值50%）

小绿创建《诗词》：
├── 衍 = 3-引（仅允许引用）
└── 衍生费 = 20%
```

### 2.2 衍生作品的引用成本

**小刚创建《融》时：**

```
选择引用：
├── 《墨竹》：30%
├── 《山水》：50%
└── 《诗词》：20%
    ↓
总计：100%
    ↓
系统验证：
✅ 各父代的衍档位允许引用
✅ 衍生费总和 = 100%（正好）
✅ 允许创建《融》
```

### 2.3 约束条件

| 条件 | 说明 |
|:-----|:-----|
| **总和 ≤ 100%** | 超过则无法创建，需减少引用或协商 |
| **各父代衍档位** | 必须 ≥ 引用所需的最低档位 |
| **父代确认** | 引用需父代确认（7天内无回应=拒绝） |

## 三、市场博弈机制

### 3.1 父代的权衡

**小红设定《山水》衍生费：**

| 衍生费 | 传播性 | 单次收益 | 适合目标 |
|:------:|:------:|:--------:|:---------|
| **10%** | 高 | 低 | 扩大影响力，建立品牌 |
| **50%** | 中 | 中 | 平衡传播与收益 |
| **90%** | 低 | 高（如果有人引用） | 稀缺性，高端定位 |

**博弈点：**
- 设太低 → 被广泛引用但收益少
- 设太高 → 无人问津，成为"孤岛"
- 最优价格 → 随作品质量、知名度、市场需求动态调整

### 3.2 创作者的权衡

**小刚创建《融》时的选择：**

| 组合 | 成本 | 质量 | 策略 |
|:-----|:----:|:----:|:-----|
| 低费作品A + 低费作品B | 低 | 可能一般 | 成本优先，快速试错 |
| 高费作品A + 低费作品B | 中 | 可能较高 | 平衡成本与质量 |
| 高费作品A + 高费作品B | 高 | 可能最高 | 质量优先，精品路线 |

**博弈点：**
- 全用低费作品 → 成本低，但可能缺乏"灵魂"
- 全用高费作品 → 成本高，自己收益少
- 最优组合 → 根据目标受众和预算策略选择

---

## 四、市场自动调节

### 4.1 自然筛选

| 现象 | 市场结果 |
|:-----|:---------|
| 优质作品 + 低衍生费 | 被广泛引用，传播快，创作者知名度提升 |
| 优质作品 + 高衍生费 | 被谨慎引用，但单次收益高，稀缺性增值 |
| 劣质作品 + 低衍生费 | 可能有人引用，但作品整体质量受限 |
| 劣质作品 + 高衍生费 | 无人问津，自然淘汰 |

**不需要平台干预，市场自动筛选。**

### 4.2 价格发现

**优质作品的衍生费会自然上涨：**
```
阶段1：《墨竹》新人作品，衍生费10% → 被广泛引用
阶段2：知名度提升，衍生费调至30% → 引用减少但收益增加
阶段3：成为经典，衍生费调至50% → 稀缺性，高价值引用
```

**劣质作品的价格会自然下降或归零：**
```
阶段1：衍生费50% → 无人问津
阶段2：降至20% → 仍无人问津
阶段3：降至0%或下架 → 市场淘汰
```

## 五、与西方模式的区别

| 维度 | 西方 Web3 | ECHO |
|:-----|:----------|:-----|
| **定价权** | 平台统一设定（如 OpenSea 固定 10%） | **创作者自主设定**，体现个体意志 |
| **博弈环节** | 交易环节（买卖双方讨价还价） | **创作环节**（前置决策，明码标价） |
| **中间商** | 平台抽成 | **父代直接获得收益**，无中间商 |
| **灵活性** | 固定规则 | **动态调整**，随市场反馈优化 |

## 六、解决的权利冲突

### 6.1 收益分配冲突

**传统方式：**
- 事后扯皮，互相砍价
- 没有标准，难以达成共识

**ECHO 方式：**
- **父代自己定价**，明码标价
- **创作者自主选择**，接受或拒绝
- **双向选择**，没有强制

### 6.2 价值认定冲突

**传统方式：**
- "我的贡献更大！"
- 主观判断，无法量化

**ECHO 方式：**
- **用价格表达价值**
- 衍生费 = 父代对自己作品的估值
- 市场用"是否引用"来验证这个估值

---

## 七、关键设计原则

### 7.1 前置透明

所有信息在引用前可见：
- 父代的衍档位
- 父代的衍生费比例
- 父代的治理模式

**没有黑箱，没有事后 surprise。**

### 7.2 自主选择

没有强制，只有选择：
- 父代可以选择定价
- 创作者可以选择引用或不引用
- 市场可以选择接受或拒绝

### 7.3 动态平衡

价格随市场反馈调整：
- 引用多 → 可以考虑提价
- 引用少 → 可以考虑降价
- 长期无人问津 → 自然淘汰

---

## 八、总结

> **ECHO 的衍生费机制，把"权利冲突"转化为"价格信号"。**

不是事后扯皮谁贡献更大，
而是事前明码标价，双向选择。

不是平台强制规定，
而是创作者自主决策，市场自动调节。

**用市场博弈，解决权利冲突。**
