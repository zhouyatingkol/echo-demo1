# ECHO 设计逻辑框架 v1.0
**从第一性原理出发，逐层推导**

---

## 第一层：问题定义

### 我们发现了什么问题？

**现状：**
- NFT 给了所有权证明，但资产是"死"的——铸造后就定格了
- 创作者失去与作品的连接——卖完就无关了
- 价值靠炒作，而非真实使用

**本质问题：**
> 数字时代，创作者失去了什么？
> 
> 不是作品，不是收益，
> **失去的是与作品的连接**，
> 是作品作为生命体的独立存在。

---

## 第二层：核心信念

### 我们相信什么？

**信念1：资产有生命**
- 数字资产可以有自己的"命、气、时、缘"
- 不是静态的拥有，而是动态的生成关系

**信念2：权利即生态**
- 权利不是边界，是生长的条件
- 如同生命需要土壤、阳光、水分、空气

**信念3：生生不息**
- 资产的理想状态不是"被拥有"，而是"在生长"
- 创作者永远与作品相连（署名权永在）

## 第三层：概念设计

### 3.1 如何让资产有生命？

**四要素模型：**
| 要素 | 对应 | 含义 |
|:----:|:-----|:-----|
| **命** | 四权力配置 | 资产的先天性格 |
| **气** | 气数 | 当下的能量状态 |
| **时** | 六爻变势 | 生命的阶段 |
| **缘** | 使用记录 | 与他人的关系 |

### 3.2 四权力框架（为什么是这四个？）

**从"使用"到"生长"的完整链条：**

```
用（使用权）—— 基础
    ↓
扩（扩展权）—— 传播
    ↓
衍（衍生权）—— 创造
    ↓
益（收益权）—— 回馈
```

**为什么是这四个？**
1. **用** —— 没有使用权，资产不存在（土壤）
2. **扩** —— 不被看见，无法生长（阳光）
3. **衍** —— 不被改编，无法演化（水分）
4. **益** —— 没有回馈，无法持续（空气）

**权力依赖：**
- 用 = 0 时，扩、衍自动锁定（没有土壤，阳光水分无从谈起）
- 这是存在论的事实，不是技术限制

### 3.3 六爻变势（为什么是六爻？）

**乾卦六爻 = 生命的六个阶段：**

| 爻位 | 名称 | 状态 | 含义 |
|:----:|:-----|:-----|:-----|
| 初九 | 潜龙勿用 | 潜藏 | 刚创建，尚未被看见 |
| 九二 | 见龙在田 | 显现 | 开始被访问、被使用 |
| 九三 | 终日乾乾 | 勤勉 | 持续被使用，积累价值 |
| 九四 | 或跃在渊 | 试探 | 出现引用、讨论、改编 |
| 九五 | 飞龙在天 | 大成 | 形成生态，广泛传播 |
| 上九 | 亢龙有悔 | 转化 | 盛极而变，准备新生 |

**为什么是六爻？**
- 《周易》乾卦的传统六爻，有经典依据
- 六个阶段覆盖了"生、长、盛、衰、藏、再生"的完整周期
- 用九（六爻皆变）触发卦象转化，进入新周期

**关键设计：不是算法决定，是真实使用驱动**
- 初九→九二：首次被访问
- 九三：7天内使用≥3次
- 九四：出现引用/衍生
- 九五：形成生态（≥5个衍生作品）
- 上九：活跃度衰减或社区投票

### 3.4 气数（为什么需要气数？）

**气数 = 当下的能量状态可视化**

**不是：**
- 随机生成的数值
- 预测未来的算法

**而是：**
- 使用频率、社区活跃度、衍生数量、时间衰减的综合体现
- 链上数据的可视化呈现

**为什么需要？**
- 给用户一个"感知资产生命状态"的方式
- 不是精确的数值，是诗意的表达

## 第四层：实现逻辑

### 4.1 技术实现原则

**原则1：记录生命，而非计算生命**
- 变势不由算法决定，由真实使用证明触发
- 气数不是随机生成，是链上数据的可视化

**原则2：去中心化验证**
- 使用证明需多节点共识（防止刷量）
- 权力配置上链锁定（不可篡改）

**原则3：渐进开放**
- 核心团队长期主导（保证方向）
- 社区逐步参与（避免低效治理）

### 4.2 商业模式逻辑

**收入来源：**
1. **铸造费** —— 创建资产（覆盖网络成本）
2. **交易手续费** —— 资产流转（平台收入）
3. **企业定制** —— 博物馆、画廊（早期造血）

**支付货币：**
- 小额高频：ECHO（Gas费用，锚定DCEP）
- 大额交易：DCEP（资产交易，合规稳定）

**为什么需要 ECHO？**
- 计量资源使用（防止 spam）
- 网络运营的"气"（区别于外部的"钱"）

## 第五层：文化定位

### 5.1 我们的立场

**不是：**
- 严格的易学学术研究
- 传统的术数应用

**而是：**
- 受东方文化启发的设计
- 为数字时代的创作服务

### 5.2 必须尊重的底线

| 要素 | 尊重什么 | 为什么 |
|:-----|:---------|:-------|
| 六爻 | 命名和顺序 | 经典依据，文化根基 |
| 群龙无首 | 本义（大成之境） | 避免误解，诚实表达 |
| 引用 | 准确性 | 学术诚信 |

### 5.3 可以创新的空间

| 要素 | 创新什么 | 为什么 |
|:-----|:---------|:-------|
| 卦象视觉 | 呈现方式 | 设计表达，非经典本身 |
| 气数算法 | 数据来源和可视化 | 应用需求，非预测 |
| 四权力档位 | 具体划分 | 实用需求，非经典依据 |

## 第六层：价值主张

### 一句话

> **"让数字资产像生命一样生长，让创作者与作品永远相连。"**

### 对谁说？说什么？

| 受众 | 痛点 | ECHO 解决什么 |
|:-----|:-----|:--------------|
| **创作者** | 作品卖完就失去连接 | 署名权永在，收益持续，看到作品生长 |
| **收藏者** | 买了 NFT 只能放着 | 资产有生命状态，可以参与生长 |
| **机构** | 数字文物无法管理 | 四权力可控，合规展示，追踪使用 |
