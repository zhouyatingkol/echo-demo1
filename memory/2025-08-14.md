## 四权配置面板重构

### 设计方向调整
Founder 提出优化方向：东方美学，温和古朴，留白呼吸感。

### 关键变化

**1. 留白增加**
- 组间距：1rem → 2.5rem
- 卡片内边距：1rem → 1.5rem
- 整体呼吸感显著增强

**2. 视觉温和化**
- 选中态：深底白字 → 透明底+金色左边线
- 边框：1px → 0.5px（更克制）
- 圆角：4px → 2px
- 移除所有投影

**3. 文案古朴化**
| 现代 | 古朴 |
|:---|:---|
| 一次性深度交换 | 一次交付，恒久留存 |
| 所有权转移 | 根性永在，铸者长名 |
| 韵律式交换 | 按需取用，次次清明 |
| 小范围·个人空间 | 一隅之地·潜心静守 |
| 铸者独拥全部 | 一唯一独，唯有铸者 |

**4. 交互舒缓**
- 过渡时间：0.3s → 0.6-0.8s
- 使用柔和缓动曲线
- 悬停效果更含蓄

### 结构层次
```
◉ 核心配置（金色标记）
◇ 辅助配置（灰色标记）
─── 概念说明（虚线分隔）
```

## realm-create.html 点击问题修复

### 问题
四象限卡片点击无法选中，事件绑定不可靠。Founder 测试多次后表达 frustration。

### 原因分析
- `addEventListener` 动态绑定时机问题
- 面板展开后才绑定事件，但绑定标记 `dataset.bound` 逻辑复杂
- 可能存在事件委托冲突

### 解决方案
**彻底重构为内联 `onclick`**

```html
<!-- 旧方式：动态绑定，不可靠 -->
<div class="card" data-value="perpetual">...</div>

<!-- 新方式：内联 onclick，最可靠 -->
<div class="card" data-value="perpetual" onclick="selectCard(this, 'yong')">...</div>
```

### 代码简化
- 行数：~800 → ~500
- 移除复杂的事件绑定逻辑
- 全局函数直接处理状态

### 教训
> 简单比优雅更重要。当动态绑定出问题，直接用内联事件。
